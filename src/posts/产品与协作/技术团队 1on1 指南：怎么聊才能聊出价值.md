# 技术团队 1on1 指南：怎么聊才能聊出价值

1on1 是管理者与团队成员最重要的沟通工具，但很多 1on1 变成了「项目进度汇报」或「没事就取消」。本文从话题清单到不同阶段策略，帮你把 1on1 变成团队成长的加速器。

---

## 一、1on1 的核心原则

```
1on1 不是什么：
  ❌ 项目进度汇报（那是站会的事）
  ❌ 绩效面谈（那是季度评审的事）
  ❌ 批评教育（那是即时反馈的事）
  ❌ 没事就取消（那说明你不重视人）

1on1 是什么：
  ✅ 专属的成长对话时间
  ✅ 倾听员工的想法和困难
  ✅ 帮助员工排除障碍
  ✅ 建立信任关系

核心原则：
  1. 员工主导（让员工说 80%，管理者说 20%）
  2. 定期坚持（每周或每两周一次，30-45 分钟）
  3. 不取消（取消 1on1 = 传递「你不重要」的信号）
  4. 跟进（上次聊的问题这次要跟进）
```

## 三、不同阶段员工的 1on1 策略

```
按阶段区分策略：

  新人（入职 0-3 个月）：
    频率：每周 1 次
    话题重点：
      - 适应情况（团队/流程/工具）
      - 入职任务进展
      - 遇到的困难
      - 第一印象反馈（新人的视角很宝贵）
    注意：
      - 多引导（新人可能不敢说问题）
      - 明确期望（让新人知道怎么才算做好）

  骨干（入职 6 个月-2 年）：
    频率：每两周 1 次
    话题重点：
      - 技术成长方向
      - 想承担更多什么责任？
      - 对项目/架构的想法
      - 职业规划
    注意：
      - 给空间（让骨干主导话题）
      - 挑战（设置更高的期望）
      - 认可（及时肯定贡献）

  资深（2 年以上）：
    频率：每月 1 次
    话题重点：
      - 技术影响力（分享/文档/带人）
      - 职业瓶颈
      - 团队方向建议
      - 管理意愿（是否想转管理）
    注意：
      - 尊重经验（多听少说）
      - 赋能（提供平台和资源）
      - 关注留任（资深离职影响大）
```

## 四、1on1 常见误区与对策

```
误区 1：变成项目汇报
  症状：每次都在聊「这个任务进度怎么样」
  对策：前 5 分钟可以同步项目，之后切到成长话题

误区 2：变成批评会
  症状：每次都在说「你哪里做得不好」
  对策：批评应该即时反馈，不要攒到 1on1 说
        1on1 应该聚焦「怎么帮你做得更好」

误区 3：管理者说太多
  症状：管理者说了 70%，员工只说了 30%
  对策：管理者说 20%，员工说 80%
        用提问引导：「你觉得呢？」「你怎么看？」

误区 4：无后续行动
  症状：聊完就完了，下次还是聊同样的问题
  对策：每次记录 Action Items
        下次开始先跟进上次的 Action Items

误区 5：取消 1on1
  症状：「太忙了，这次取消吧」
  对策：1on1 是最重要的管理动作，不取消
        如果真的忙，缩短到 15 分钟也比取消好
        取消 = 传递「你不重要」的信号
```

## 六、面试要点

### Q：怎么做 1on1？

1on1 是管理者和员工的专属成长对话，不是项目汇报。核心原则：员工说 80%、管理者说 20%；定期坚持不取消；跟进上次 Action Items。话题覆盖职业发展、技术成长、团队协作、痛点反馈、心理状态。不同阶段策略不同：新人每周聊适应，骨干每两周聊成长，资深每月聊方向。

### Q：1on1 取消有什么问题？

取消 1on1 传递「你不重要」的信号，破坏信任关系。如果真的忙，缩短到 15 分钟也比取消好。1on1 是管理者最重要的投资——半小时的倾听可以避免几周的绩效问题。

## 七、总结

```
1on1 = 员工主导 + 定期坚持 + 跟进闭环

原则：员工说 80% / 不取消 / 不变成汇报会 / 跟进 Action Items
话题：职业发展 / 技术成长 / 团队协作 / 痛点反馈 / 心理状态
策略：新人（适应/每周）/ 骨干（成长/每两周）/ 资深（方向/每月）
记录：Action Items + 下次跟进
```
